// 01 — PRESENTACIÓN

¿Qué es MaPoint?

MaPoint es un programa para Windows que hace de centro de mando para un robot agrícola que se mueve solo. El robot se conecta al ordenador por WiFi: le manda su posición GPS en tiempo real, y el programa le dice por dónde ir.

El operador marca puntos en un mapa por satélite y el robot los recorre automáticamente, esquivando obstáculos con la cámara. Si lleva la compuerta de semillas, puede programarla para abrirse y cerrarse en coordenadas concretas del campo.

// 02 — SUBSISTEMAS

¿Qué hace el sistema?

🛰️
GPS EN TIEMPO REAL

El robot envía su posición cada segundo. El programa la recibe, calcula a qué distancia está del siguiente punto y en qué dirección tiene que girar.

🗺️
MAPA POR SATÉLITE

El operador hace clic en el mapa para marcar los puntos que quiere que el robot visite. La ruta se dibuja en tiempo real y el robot aparece moviéndose sobre el mapa.

🤖
DETECCIÓN DE OBSTÁCULOS

La cámara analiza lo que tiene delante dividiendo la imagen en tres zonas. Si detecta algo en el camino, decide esquivarlo automáticamente por la izquierda o la derecha.

📡
VÍDEO EN DIRECTO

Muestra la imagen de la cámara del robot con información superpuesta: qué decisión está tomando la IA y a cuántos fotogramas por segundo va la señal.

🌱
ANÁLISIS DE SUELO

Analiza el color y la textura del suelo visible en la cámara para saber si es buen momento para plantar. Detecta humedad, vegetación ya existente o suelo demasiado seco.

⚙️
COMPUERTA DE SEMILLAS

Puedes programar en qué punto del campo se abre y se cierra la compuerta. El robot lo hace solo cuando llega a esa coordenada GPS durante la ruta.

📊
INFORMES AL TERMINAR

Al acabar la misión, genera automáticamente un informe de texto, una hoja de Excel con estadísticas y un archivo para ver la ruta exacta en Google Earth.

🎮
CONTROL MANUAL

Con las flechas del teclado puedes mover el robot tú mismo cuando quieras. La tecla M centra la dirección. Puedes cambiar entre modo manual y autónomo en cualquier momento.

// 03 — ARQUITECTURA

¿Cómo funciona por dentro?

MaPoint hace tres cosas a la vez sin que se bloqueen entre sí. Es como tener tres personas trabajando en paralelo: una mirando el GPS, otra viendo la cámara, y una tercera decidiendo hacia dónde ir. El programa las coordina para que la pantalla siempre responda fluidamente.

01
RECIBE EL GPS

Consulta la posición del robot por WiFi cada 0,8 segundos y actualiza el punto en el mapa.

02
VE LA CÁMARA

Recibe el vídeo del robot, lo analiza para buscar obstáculos y muestra todo en pantalla.

03
DECIDE EL RUMBO

Compara dónde está el robot con el siguiente punto de la ruta. Si está de lado, primero gira; si está cerca (menos de 2 metros), pasa al siguiente punto.

04
ENVÍA ÓRDENES

Manda al robot mensajes de texto por WiFi: "adelante", "izquierda", "parar"... El ESP32 los recibe y mueve los motores.

05
ACTUALIZA LA PANTALLA

Todo se actualiza cada 50 milisegundos para que la interfaz no se congele mientras el robot trabaja.

06
GENERA EL INFORME

Al terminar, crea el Excel, el informe de texto y el archivo de Google Earth con la ruta recorrida.

// 04 — GUÍA TÉCNICA

Guía de Montaje y Programación

Aquí te explicamos, paso a paso y en lenguaje sencillo, cómo montar el robot y programarlo para que funcione con MaPoint. El robot es de tracción trasera: dos motores mueven las ruedas de atrás, y un servo gira las ruedas delanteras como la dirección de un coche. El GPS Adafruit le dice dónde está, la ESP32-CAM graba lo que ve, y la compuerta (opcional) siembra de forma automática. Sigue las pestañas en orden la primera vez.

⚠ IMPORTANTE ANTES DE EMPEZAR

El GPS Adafruit funciona a 3,3 voltios. Si le conectas 5 voltios directamente, se rompe. El L298N sí necesita la tensión completa de la batería (7–12 V) para los motores, pero sus pines de señal se conectan al ESP32 sin problema.

Vamos a programar el ESP32 paso a paso. Si nunca has usado el Arduino IDE con ESP32, empieza por el paso 0. Si ya lo tienes configurado, ve al paso 1.

El programa que vamos a cargar hace tres cosas: crea una red WiFi a la que se conectará el ordenador, comparte la posición GPS del robot por esa red, y escucha las órdenes de movimiento que envía MaPoint para mover los motores y el servo.

00 INSTALAR EL ARDUINO IDE Y AÑADIR EL ESP32

Si ya tienes Arduino IDE con el ESP32 instalado, sáltate este paso.

1. Descarga e instala Arduino IDE desde arduino.cc.

2. Abre Arduino IDE. Ve a Archivo → Preferencias y en "Gestor de URLs adicionales de tarjetas" pega esta dirección exacta:

https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

3. Ve a Herramientas → Placa → Gestor de tarjetas, busca "esp32" e instala el de Espressif Systems.

4. Conecta el ESP32 al ordenador con un cable USB. En Herramientas → Placa selecciona "ESP32 Dev Module". En Puerto elige el COM que aparezca.

5. Instala la librería del servo: Herramientas → Gestionar librerías, busca "ESP32Servo" de Kevin Harrington e instálala.

01 CABECERA: LIBRERÍAS Y PINES

Crea un nuevo sketch en Arduino IDE. Copia y pega este bloque al principio. Aquí definimos qué herramientas usar y a qué pines hemos conectado cada componente. Si cambiaste algún pin al montar, cámbialo aquí también.

#include <WiFi.h>           // Para crear la red WiFi del robot
#include <WebServer.h>      // Para publicar el GPS en una dirección web
#include <WiFiUdp.h>        // Para recibir órdenes desde MaPoint
#include <HardwareSerial.h> // Para leer el GPS por cable
#include <ESP32Servo.h>     // Para mover el servo de dirección

// ── Red WiFi que crea el robot ─────────────────────────────
// MaPoint se conectará a este nombre y contraseña
const char* SSID     = "MaPoint-Robot";
const char* PASSWORD = "mapoint123";

// ── Pines del driver L298N (motores traseros) ──────────────
#define ENA  13   // Velocidad del motor (más valor = más rápido)
#define IN1  26   // Controla el sentido de giro
#define IN2  27   // Controla el sentido de giro

// ── Pines de los servos ────────────────────────────────────
#define PIN_SERVO_DIR  25  // Servo de dirección (ruedas delanteras)
#define PIN_SERVO_COMP 33  // Servo compuerta semillas (opcional)

// ── Pin del GPS ────────────────────────────────────────────
#define GPS_RX 16  // El ESP32 escucha aquí los datos del GPS
#define GPS_TX 17  // Para hablar con el GPS (casi no se usa)

// ── Objetos internos ───────────────────────────────────────
HardwareSerial gpsSerial(2); // UART2 del ESP32 para el GPS
WebServer server(80);        // Servidor web en el puerto 80
WiFiUDP   udp;               // Para recibir órdenes de MaPoint
Servo servoDir;              // Servo de dirección
Servo servoComp;             // Servo compuerta (opcional)

// ── Variables internas ─────────────────────────────────────
String nmeaBuffer = ""; // Aquí guardamos los datos GPS antes de enviarlos
String lineaGPS   = ""; // Línea actual que llega del GPS
const int PWM_VEL  = 200; // Velocidad (0 = parado, 255 = máximo)
const int UDP_PORT = 8888; // Puerto donde escuchamos las órdenes
02 MOVER LOS MOTORES Y EL SERVO DE DIRECCIÓN

Estas son las funciones que mueven el robot. El L298N controla los motores traseros (los que dan la tracción), y el servo gira las ruedas delanteras para cambiar de dirección.

El servo trabaja con ángulos: 90° = recto, 55° = izquierda, 125° = derecha. Si el servo gira al revés de lo que esperabas, simplemente cambia el valor de izquierda por el de derecha y viceversa.

// ── MOTORES TRASEROS ──────────────────────────────────────

void motorParar() {
  digitalWrite(IN1, LOW);
  digitalWrite(IN2, LOW);
  analogWrite(ENA, 0);
}

void motorAdelante(int vel = PWM_VEL) {
  digitalWrite(IN1, HIGH);
  digitalWrite(IN2, LOW);
  analogWrite(ENA, vel);
}

void motorAtras(int vel = PWM_VEL) {
  digitalWrite(IN1, LOW);
  digitalWrite(IN2, HIGH);
  analogWrite(ENA, vel);
}

// ── SERVO DE DIRECCIÓN (cremallera) ───────────────────────
// Ajusta los ángulos si el giro no coincide con tu montaje

void dirRecto()      { servoDir.write(90);  } // Recto
void dirIzquierda()  { servoDir.write(55);  } // Giro izquierda
void dirDerecha()    { servoDir.write(125); } // Giro derecha

// ── COMPUERTA DE SEMILLAS (opcional) ──────────────────────
void abrirCompuerta()  { servoComp.write(90); } // Abierta
void cerrarCompuerta() { servoComp.write(0);  } // Cerrada

Consejo: Si el robot gira menos de lo que debería, prueba con ángulos más extremos, por ejemplo 40° y 140°.

03 RECIBIR LAS ÓRDENES QUE ENVÍA MAPOINT

MaPoint manda mensajes de texto por WiFi para decirle al robot qué hacer: "adelante", "izquierda", "parar"... Esta función los recibe y llama a la acción correspondiente.

Por ejemplo, cuando MaPoint dice "izquierda", el servo gira a la izquierda y el motor avanza a la vez, igual que un coche normal.

void procesarOrdenes() {
  int len = udp.parsePacket();
  if (!len) return; // si no hay mensaje, salir

  char buf[64];
  udp.read(buf, sizeof(buf) - 1);
  buf[len] = '\0';
  String orden = String(buf);
  orden.trim(); // eliminar espacios sobrantes

  // Mostrar en el monitor serie para saber qué pasa
  Serial.println("Orden recibida: " + orden);

  if (orden == "adelante") {
    dirRecto(); motorAdelante();
  } else if (orden == "atras") {
    dirRecto(); motorAtras();
  } else if (orden == "izquierda") {
    dirIzquierda(); motorAdelante();
  } else if (orden == "derecha") {
    dirDerecha(); motorAdelante();
  } else if (orden == "parar") {
    motorParar(); dirRecto();
  } else if (orden == "centrar_direccion") {
    // Tecla M en MaPoint: para y centra las ruedas
    motorParar(); dirRecto();
  } else if (orden == "abrir_compuerta") {
    abrirCompuerta();
  } else if (orden == "cerrar_compuerta") {
    cerrarCompuerta();
  }
}
04 LEER EL GPS Y COMPARTIR LA POSICIÓN

El GPS manda continuamente pequeños mensajes de texto con la posición (son las llamadas "tramas NMEA"). El ESP32 los va guardando y los publica en una dirección web, de forma que MaPoint puede consultarla cada segundo para saber dónde está el robot.

// Acumula los mensajes que llegan del GPS ──────────────────
void leerGPS() {
  while (gpsSerial.available()) {
    char c = (char)gpsSerial.read();
    if (c == '\n') {
      lineaGPS.trim();
      if (lineaGPS.startsWith("$")) {
        nmeaBuffer += lineaGPS + "\n";
        // Limite de 2 KB para no llenar la memoria
        if (nmeaBuffer.length() > 2000) nmeaBuffer = "";
      }
      lineaGPS = "";
    } else {
      lineaGPS += c;
    }
  }
}

// Publica la posición GPS en una URL ──────────────────────
// MaPoint consulta http://192.168.4.1/gps para obtenerla
void configurarServidor() {
  server.on("/gps", HTTP_GET, []() {
    server.send(200, "text/plain", nmeaBuffer);
    nmeaBuffer = ""; // limpiar después de enviar
  });
  server.begin();
}
05 SETUP Y LOOP — EL PROGRAMA COMPLETO

El setup() se ejecuta una sola vez al encender: configura los pines, crea la red WiFi y arranca el servidor. El loop() se repite sin parar: lee el GPS, responde si MaPoint pide datos, y ejecuta las órdenes de movimiento.

void setup() {
  Serial.begin(115200); // Para ver mensajes en el monitor serie

  // Iniciar comunicación con el GPS a 9600 baudios
  gpsSerial.begin(9600, SERIAL_8N1, GPS_RX, GPS_TX);

  // Configurar pines del L298N como salida
  pinMode(ENA, OUTPUT);
  pinMode(IN1, OUTPUT);
  pinMode(IN2, OUTPUT);
  motorParar(); // Empezar con el robot parado (por seguridad)

  // Iniciar los servos
  servoDir.attach(PIN_SERVO_DIR);
  servoComp.attach(PIN_SERVO_COMP); // quitar esta línea si no tienes compuerta
  dirRecto();        // Ruedas al centro al arrancar
  cerrarCompuerta(); // Compuerta cerrada al arrancar

  // Crear la red WiFi del robot
  WiFi.softAP(SSID, PASSWORD);
  Serial.print("Robot listo. IP: ");
  Serial.println(WiFi.softAPIP()); // Debe mostrar 192.168.4.1

  // Publicar el GPS y empezar a escuchar órdenes
  configurarServidor();
  udp.begin(UDP_PORT);
  Serial.println("Esperando conexión de MaPoint...");
}

void loop() {
  leerGPS();              // Guardar datos del GPS
  server.handleClient();  // Responder si MaPoint pide la posición
  procesarOrdenes();      // Mover el robot si llega una orden
}

Para subir el código: pulsa la flecha (→) en Arduino IDE. Si da error de puerto, asegúrate de que el ESP32 está conectado por USB y el puerto COM correcto está seleccionado en Herramientas → Puerto. Abre el Monitor Serie a 115200 baudios para ver los mensajes de arranque.

La ESP32-CAM es una cámara con WiFi incorporado. No hace falta escribir código desde cero: Arduino IDE ya incluye un ejemplo llamado CameraWebServer que hace exactamente lo que necesitamos. Solo hay que cambiar dos líneas para que la cámara se conecte a la red WiFi del robot en vez de a tu router de casa.

01 ABRIR EL EJEMPLO DE CÁMARA

En Arduino IDE ve a: Archivo → Ejemplos → ESP32 → Camera → CameraWebServer.

Al principio del código verás varias líneas con nombres de modelos de cámara. La mayoría estarán comentadas (con // delante). Deja solo esta sin comentar y comenta el resto:

#define CAMERA_MODEL_AI_THINKER   // ← esta sí, el resto comentadas
// #define CAMERA_MODEL_WROVER_KIT
// #define CAMERA_MODEL_ESP_EYE

También necesitas seleccionar la placa correcta: Herramientas → Placa → "AI Thinker ESP32-CAM".

02 CAMBIAR EL WIFI AL DEL ROBOT

Busca en el archivo estas dos líneas (están al principio) y cámbialas por el nombre y contraseña de la red del robot, las mismas que pusiste en el ESP32 principal:

const char* ssid     = "MaPoint-Robot"; // igual que en el ESP32
const char* password = "mapoint123";    // igual que en el ESP32

Así la cámara se conectará automáticamente a la red del robot cuando arranque, y MaPoint la encontrará en http://192.168.4.x:81/stream.

03 CÓMO SUBIR EL CÓDIGO A LA ESP32-CAM

La ESP32-CAM no tiene puerto USB propio, así que necesitas un adaptador USB-UART (también llamado FTDI o CH340). Son muy baratos.

Pasos:

1. Conecta el cable de GND al pin IO0 (modo de programación).

2. Conecta el FTDI al ordenador y sube el código desde Arduino IDE.

3. Cuando termine, desconecta el cable de IO0.

4. Pulsa el botón RESET de la ESP32-CAM.

5. Abre el Monitor Serie a 115200 baudios. Verás algo como: Camera Ready! Use 'http://192.168.4.2' to connect. Esa dirección es la que MaPoint usará para el vídeo.
